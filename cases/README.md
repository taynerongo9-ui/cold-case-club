## File Structure

Each case folder contains:
```
case-N-name/
├── case-bible.md          # Complete story bible — DO NOT SHARE WITH CUSTOMERS
│                          # Contains: full narrative, solution, character profiles,
│                          # timeline, clue map, production notes
└── packets/
    ├── packet-01.md       # Evidence packet 1 (mailed at start)
    ├── packet-02.md       # Evidence packet 2 (mailed 2 weeks later)
    ├── ...
    └── packet-12.md       # Final packet — solution reveal
```

## Production Notes

- Each evidence piece has `[PRODUCTION NOTE]` or `[PRINT FORMAT]` brackets with physical format guidance
- Paper stock, font, aging effects, and artifact suggestions included per piece
- Coded messages use real ciphers that subscribers can solve (Caesar, substitution, etc.)
- All dates, names, and details are internally consistent per case bible
